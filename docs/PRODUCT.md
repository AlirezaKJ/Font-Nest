# Product

## Register

product

## Users

FontNest is for designers, typographers, frontend engineers, and other typography-focused professionals who need to understand the fonts available on their computer without moving between several tools.

Their work includes inventorying installed fonts, evaluating families and faces, checking weights and styles, inspecting variable axes and glyph coverage, diagnosing duplicate or conflicting files, and safely managing fonts they intentionally installed through FontNest. They may work with thousands of faces, so the interface must remain fast, legible, and trustworthy during sustained inspection sessions.

## Product Purpose

FontNest gives users one dependable desktop workspace for inspecting and managing local fonts. It combines a searchable catalogue, accurate live previews, detailed font metadata, duplicate detection, and guarded install or uninstall operations.

Success means users can quickly answer three questions: what fonts are installed, which font is right for the work in front of them, and whether any files are duplicated or conflicting. The application should make routine inspection effortless while making destructive operations deliberately safe.

## Brand Personality

Precise, calm, and discerning.

FontNest should feel professional and approachable without becoming decorative, nostalgic, or artificially cozy. Its voice is concise and confident. It explains font and system details clearly, respects the user's expertise, and becomes quieter as the user's library becomes more information-dense.

## Anti-references

FontNest should not resemble a generic SaaS analytics dashboard, a novelty font marketplace, or an overdecorated creative tool whose branding competes with the user's typefaces. Avoid gratuitous cards, ornamental typography in controls, excessive motion, decorative glass effects, and unfamiliar interaction patterns where standard desktop affordances would be clearer.

The interface must never make system fonts appear casually removable, hide the difference between system and app-managed files, or use visual charm to soften a destructive action.

## Design Principles

1. **Let the fonts be the expressive material.** The application frame stays composed and restrained so font previews, glyphs, and metadata remain the focus.
2. **Make inspection immediate.** Cached information appears quickly, search and filters respond without friction, and deeper metadata is progressively disclosed only when requested.
3. **Earn trust through explicit boundaries.** System fonts, app-managed installations, duplicates, conflicts, and destructive actions are labelled unambiguously.
4. **Support expert depth without expert-only navigation.** Common tasks remain obvious while advanced metadata, variable axes, glyph coverage, and file details are available without cluttering the library.
5. **Stay calm at catalogue scale.** Dense libraries remain readable and stable through consistent controls, virtualization, restrained motion, and strong information hierarchy.

## Accessibility & Inclusion

FontNest targets WCAG 2.2 AA as its minimum accessibility baseline. Every workflow must be fully keyboard-operable, with visible focus, logical navigation order, and no status conveyed by color alone. Text and interactive controls must maintain compliant contrast at all supported themes and states.

Motion must respect `prefers-reduced-motion` and communicate state rather than decorate the interface. Font previews must not compromise the readability of labels or controls, and users must always be able to fall back to a stable system UI font. The interface should remain comfortable during long inspection sessions and under text scaling, high-contrast settings, and common forms of color-vision deficiency.
